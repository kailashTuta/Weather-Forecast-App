# Ontology Schema Generation Prompt

## Date
Project Initialization

## Prompt

Generate a JSON-LD schema for an AngularJS-to-ReactJS Migration Ontology that includes application components, AngularJS modules, ReactJS components, services, controllers, directives, routing, state management, APIs, dependencies, UI elements, and migration strategies. Include requirement categories such as functional, non-functional, security, performance, scalability, and maintainability. Define stakeholders such as frontend developers, architects, QA engineers. Include entities for business goals, migration phases, use cases, risks, compatibility constraints, and test cases. Add properties to represent relationships such as component mappings, dependencies, conflicts, replacements, verification methods, migration status, and test coverage. Include support for tracking deprecated AngularJS features and their equivalent ReactJS implementations

## Context

This was the initial prompt used to create the comprehensive ontology schema for the AngularJS to React migration project. The schema serves as the foundation for documenting and tracking the entire migration process.

## Result

Created `angularjs-to-react-migration-ontology.jsonld` file containing:
- Complete ontology structure with classes and properties
- Migration tracking entities
- Requirement categories and relationships
- Component mapping definitions
- Stakeholder roles and responsibilities
- Test coverage and verification methods
